# PureTones ‚Äì Real-Time Brainwave & Breath Tone Generator

PureTones is a small Python tool that generates **pure tones for focus, healing, breath regulation, and relaxation**.

It works entirely **offline**, generating audio mathematically in real time and sending it directly to your output device (e.g. AirPods Max).
No files are streamed, no microphone is used, and nothing external is mixed in.

### Untamperable by Design

Every sample that reaches your ears is **mathematically computed in-process** ‚Äî there is no way to inject, mix, or substitute external audio:

- **Single output stream only** ‚Äî one `sounddevice.OutputStream`, no input, no duplex, no second stream
- **No external sources** ‚Äî no TTS, no file playback, no OS voice synthesis, no network audio
- **No input devices** ‚Äî microphone access is architecturally impossible (not just disabled ‚Äî never opened)
- **Deterministic** ‚Äî fixed RNG seed, reproducible across runs, no runtime entropy
- **Cryptographic integrity proof** (`--integrity`) ‚Äî rolling SHA-256 hash of every generated buffer
- **Lockdown mode** (`--lockdown`) ‚Äî pure sine + output-only + integrity proof combined

The audio callback is the only code that writes to the output buffer. What you hear is exactly what was computed.

Breath Cues & Breathing Bar

Breath cues are short synthesized sounds that mark inhale/exhale transitions during HRV mode. They are pre-computed at startup as NumPy arrays ‚Äî never loaded from files, never generated by TTS.

Available cues:
	‚Ä¢	bell ‚Äì Multi-partial singing bell (400ms, 5 inharmonic partials)
	‚Ä¢	bowl ‚Äì Singing bowl sustain (600ms, slow decay)
	‚Ä¢	drum ‚Äì Low thump with body resonance (200ms)
	‚Ä¢	waterdrop ‚Äì Descending chirp (250ms)
	‚Ä¢	whoosh ‚Äì Soft filtered air (500ms, 3-pass low-pass)
	‚Ä¢	tick ‚Äì Short click (30ms)
	‚Ä¢	doubletick ‚Äì Two quick clicks
	‚Ä¢	woodblock ‚Äì Percussive tap (60ms)

Inhale cues play at normal pitch; exhale cues are pitched down (0.85x) for an audible "in = higher, out = lower" distinction.

The breathing bar (--breath-bar) shows a colored terminal display with sinusoidal fill that matches the auditory envelope, elapsed time, and cycle count.

python stream_tone.py --freq 432 --hrv --breath-cue bowl --breath-bar
